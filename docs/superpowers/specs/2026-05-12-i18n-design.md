# i18n (Tiếng Việt / Tiếng Anh) — Design Spec

## Goal

Thêm hỗ trợ đa ngôn ngữ Tiếng Việt và Tiếng Anh cho toàn bộ frontend Concord. User tự chọn ngôn ngữ qua toggle `VI / EN` trên header; preference lưu vào cookie `NEXT_LOCALE` (tồn tại 1 năm). URL không thay đổi.

---

## Architecture

### Library

`next-intl` v4 — tối ưu cho Next.js 16 App Router, hỗ trợ cả server và client components, TypeScript-safe.

### Locales

- `vi` — Tiếng Việt (default, source of truth)
- `en` — Tiếng Anh

### File structure mới

```
apps/web/
├── messages/
│   ├── vi.json          # Toàn bộ strings tiếng Việt
│   └── en.json          # Toàn bộ strings tiếng Anh
├── i18n/
│   ├── routing.ts       # locales list, defaultLocale
│   └── request.ts       # next-intl config, load messages per request
└── middleware.ts        # đọc cookie NEXT_LOCALE, set locale header
```

### Routing

Không dùng URL prefix (`/vi/`, `/en/`). URL giữ nguyên hiện tại. Locale được resolve qua:
1. Cookie `NEXT_LOCALE` (priority cao nhất)
2. Default `vi` nếu cookie không có hoặc không hợp lệ

### Middleware

`middleware.ts` dùng `createNextIntlMiddleware` với `localeDetection: false` (tắt browser language detection) và `localePrefix: 'never'`. Chỉ đọc cookie.

---

## Toggle UI

**Component:** `apps/web/components/layout/locale-toggle.tsx` (client component mới)

**Vị trí:** Header, bên trái avatar button.

**Render:**
```tsx
<div className="flex items-center rounded-lg border border-stone-200 text-xs font-medium overflow-hidden">
  <button className={locale === 'vi' ? 'bg-emerald-50 text-emerald-700 px-2 py-1' : 'px-2 py-1 text-stone-400 hover:text-stone-600'}>VI</button>
  <button className={locale === 'en' ? 'bg-emerald-50 text-emerald-700 px-2 py-1' : 'px-2 py-1 text-stone-400 hover:text-stone-600'}>EN</button>
</div>
```

**Behavior khi click:**
1. Set cookie `NEXT_LOCALE=<locale>` với `max-age=31536000; path=/`
2. Gọi `router.refresh()` — re-render server components với locale mới, không full reload

## Usage Pattern

### Client component (tất cả pages hiện tại đều là `'use client'`)

```tsx
import { useTranslations } from 'next-intl';

export function MyComponent() {
  const t = useTranslations('common');
  return <button>{t('save')}</button>;
}
```

### Interpolation

```tsx
// vi.json: "yearly_goal_title": "Mục tiêu tiết kiệm năm {year}"
t('finance.yearly_goal_title', { year: 2026 })
// → "Mục tiêu tiết kiệm năm 2026"
```

## Không translate

- Tên riêng: `Concord`, `VND`, `JWT`
- Giá trị động từ API: `user.name`, `fund.name`, `formatVND(...)`
- Error messages từ backend (trả về tiếng Việt, giữ nguyên)
- Placeholder số và format date (dùng `Intl` theo locale)

## TypeScript Safety

next-intl v4 tự generate types từ `vi.json`. Thêm vào `tsconfig.json`:

```json
{
  "compilerOptions": {
    "paths": {
      "@/i18n/routing": ["./i18n/routing.ts"]
    }
  }
}
```

Và tạo `global.d.ts`:

```ts
import en from './messages/en.json';
type Messages = typeof en;
declare global {
  interface IntlMessages extends Messages {}
}
```

Khi đó `t('invalid.key')` sẽ báo lỗi TypeScript compile-time.

---

## Constraints

- Không thay đổi URL structure
- Không đụng backend / API
- Không thêm server components mới (giữ `'use client'` pattern hiện tại)
- Cookie `NEXT_LOCALE` không conflict với cookie auth (`concord_access_token`)
